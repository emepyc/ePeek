Why e!Peek?

  • Ensembl data is accessible through...
    • The Ensembl Genome Browser
    • The Ensembl Perl API
    • MySQL client
    • The Ensembl REST API (HTTP API)
      =>Allows access of Ensembl data directly in the client browser

Why e!Peek?

The Ensembl Genome Browser:

  • Is a rich featured resource for visualizing the Ensembl genomic data
  • It may give too much information for some users
  • It evolves slower than the web technologies
Genomic context is interesting to our users and generally they don't want to have to get the huge amount of detail that Ensembl provides.
Alex Bateman about RFAM

What is e!Peek?

  • A browser client of the Ensembl REST API
  • A way to create Lightweight, modular, alternative views of Ensembl data
  • A framework to test new developments in genomic data visualization

Extending e!Peek modularity

Building blocks:

  <script src="http://www.ebi.ac.uk/~mp/ePeek_genome.min.js"></script>
  <script src="http://www.ebi.ac.uk/~mp/ePeek_transcript.min.js"></script>
  <script src="http://www.ebi.ac.uk/~mp/ePeek_geneTree.min.js"></script>
  <script src="http://www.ebi.ac.uk/~mp/ePeek_genomeAlignment.min.js"></script>
  <script src="http://www.ebi.ac.uk/~mp/ePeek/full.min.js"></script>
  (funcion() {
     var genomeView = epeek_genome().species("human").gene(BRCA2);
     var transcriptView = epeek_transcript();
     var geneTreeView = epeek_geneTree().format("condensed").internal_nodes(true).taxon_ids(true);
     var genomeAlignmentView = epeek_genomeAlignment().query_species("mouse");
     var gBTheme = epeek_theme();
     var gBTheme(genomeView,
                 transcriptView,
                 geneTreeView,
                 genomeAlignmentView,
                 document.getElementById("genomeBrowser"));
  })();
