Genome Browsers the other way around...



  • e!Peek allows to create genome browsers that are...


    • lightweight
    • re-usable
    • targeted at specific needs (feature-oriented)
    • adaptable to different javascript environment
    • easily embeddable in any client web page
    • easy to experiment with

Genome Browsers





  • Genome Browsers typically are...

    • Heavy-weighted, fully featured web applications
    • Try to provide the most-comprehensive data available
    • Very difficult to evolve and adapt to new technologies / environments
    • Very difficult to experiment with
    • Not suitable for specific needs

e!Peek

<!-- Remote D3.js -->
<script src="http://d3js.org/d3.v3.min.js"></script>
<!-- Remote e!Peek JS -->
<script src="http://www.ebi.ac.uk/~mp/ePeek.min.js"></script>
<script src="http://www.ebi.ac.uk/~mp/ePeek/minimal.min.js"></script>
<!-- Remote e!Peek CSS -->
<link rel="stylesheet" href="http://www.ebi.ac.uk/~mp/ePeek.css" type="text/css" />

<div id="genomeBrowser"></div>

<script>
  (function() {
     var gB = epeek.genome().species("human").gene("BRCA2");
     var gBTheme = epeek_theme();
     var gBTheme(gB, document.getElementById("genomeBrowser"));
  })();
</script>
